# Host-Pointer Mount Grammar Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the host-pointer mount points consistent (`VAULT_PATH → /workspace/vault`) and give `DOCS_PATH`/`SPECS_PATH` a `[@]source[:ro|:rw]` grammar (named-volume `@name` + a docs writability suffix), with `DOCS_PATH` re-pointing to the working-dir mount when the docs repo *is* the working dir.

**Architecture:** All logic lives in `runme.sh`'s `run_container`. A `@name` pointer is *desugared* into the existing `REPOS` list (a pre-pass before the repo loop) so the repo machinery mounts it at `/workspace/<name>`; host-path pointers keep their fixed slots (`/workspace/{vault,specs,docs}`). Verified by the existing fake-`docker` integration harness (`tests/test-docs-path.sh`); registered `@name` repos are exercised with a Linux `bind`-backend registry record, which needs no real Docker volume.

**Tech Stack:** Bash (`runme.sh`, `project-init.sh`, `sandbox-common.sh`), Docker CLI, Markdown docs.

## Global Constraints

- Mount points by category — **host paths → fixed slots** `/workspace/vault`, `/workspace/specs`, `/workspace/docs`; **`@name` volumes → `/workspace/<name>`**. Copy these verbatim.
- Grammar per pointer: `DOCS_PATH=[@]src[:ro|:rw]` (default `:ro`); `SPECS_PATH=[@]src` (always `:rw`); `VAULT_PATH=src` (plain host path, `:rw`). `VAULT_PATH` does **not** accept `@name` or a suffix.
- `DOCS_PATH` re-point: when the resolved host path equals the resolved primary working-dir path (`$primary_path`), skip the `/workspace/docs` mount and re-export `DOCS_PATH=$workdir`; the suffix is ignored (a working dir is always writable). This **replaces** the `project-init.sh` `unset DOCS_PATH` behavior, which is removed.
- `@name` desugars into `REPOS`: mount at `/workspace/<name>` via the existing loop; if the name is already listed (`REPOS`/`@primary`), reuse it (existing mode wins; note a mode divergence) instead of double-mounting. An unregistered `@name` errors exactly like an unregistered `REPOS`/`@primary` entry.
- Missing host-path dir → `WARNING` and continue. Fixed-slot name (`vault`/`specs`/`docs`) already claimed by `REPOS`/`EXTRA_MOUNTS`/primary → `ERROR` and `exit 1`.
- `VAULT_PATH → /workspace/vault` is a **breaking change** (CHANGELOG breaking entry). Runtime-safe: the plugin repos read `$VAULT_PATH`, never the literal `/workspace/obsidian`.
- Edit canonical files only. `CLAUDE.md`, `.github/copilot-instructions.md`, `.kiro/steering/AGENTS.md` are symlinks to `AGENTS.md` — never edit them directly. Never hand-edit the `.ai-containers/` synced copies (regenerated by `sync-to-projects.sh`).
- Work stays on branch `host-pointer-mounts`. Commit after every task. Match the existing block style; edits are surgical.
- Bash target is 4+ (the script already uses `declare -A`). Do **not** use `local -n` namerefs (macOS brew bash is fine, but avoid the dependency) — pass arrays as positional args, as `pointer_repo_entry` does.

Insert immediately after the `source "${_here}/sandbox-common.sh"` line (before `usage()`):

```bash

# Parse a host-pointer value "[@]<source>[:ro|:rw]" into three globals the caller
# reads immediately: PTR_KIND (volume|path), PTR_SRC (repo name or host path),
# PTR_MODE (ro|rw — the trailing suffix when $3 is 1, else the $2 default).
parse_pointer_spec() {  # $1=raw value  $2=default mode  $3=allow_suffix(1|0)
  local val="$1" default_mode="$2" allow_suffix="$3"
  PTR_MODE="$default_mode"
  if [[ "$allow_suffix" == "1" ]]; then
    case "$val" in
      *:ro) PTR_MODE="ro"; val="${val%:ro}" ;;
      *:rw) PTR_MODE="rw"; val="${val%:rw}" ;;
    esac
  fi
  if [[ "${val:0:1}" == "@" ]]; then
    PTR_KIND="volume"; PTR_SRC="${val#@}"
  else
    PTR_KIND="path"; PTR_SRC="$val"
  fi
}

# For a @name pointer, echo "name:mode" to append to the repo list — UNLESS 'name'
# is already listed, in which case echo nothing (the existing REPOS/@primary entry
# wins) and note a mode divergence on stderr. Pass the current repo list as $3...
pointer_repo_entry() {  # $1=name  $2=mode  $3..=current repos_list entries
  local name="$1" mode="$2"; shift 2
  local e existing_mode
  for e in "$@"; do
    if [[ "${e%%:*}" == "$name" ]]; then
      existing_mode="${e##*:}"; [[ "$existing_mode" == "$name" ]] && existing_mode="ro"
      if [[ "$existing_mode" != "$mode" ]]; then
        printf "NOTE: repo '%s' is already mounted (%s); the pointer's :%s is ignored.\n" \
          "$name" "$existing_mode" "$mode" >&2
      fi
      return 0
    fi
  done
  printf '%s:%s' "$name" "$mode"
}
```

  # ── Host-pointer @name desugar ───────────────────────────────────────────────
  # DOCS_PATH/SPECS_PATH may name a registered repo volume (@name); treat it like a
  # REPOS entry so the loop below mounts it at /workspace/<name> (reusing an existing
  # entry instead of double-mounting). Host-path forms are handled after the loop.
  local docs_kind="" docs_src="" docs_mode=""

    if [[ "$docs_kind" == "volume" ]]; then
      _entry="$(pointer_repo_entry "$docs_src" "$docs_mode" ${repos_list[@]+"${repos_list[@]}"})"
      [[ -n "$_entry" ]] && repos_list+=("$_entry")

  if [[ -n "${SPECS_PATH:-}" ]]; then
    parse_pointer_spec "$SPECS_PATH" rw 0
    specs_kind="$PTR_KIND"; specs_src="$PTR_SRC"; specs_mode="$PTR_MODE"
