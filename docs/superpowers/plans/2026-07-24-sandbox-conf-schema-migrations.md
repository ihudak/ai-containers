# `sandbox.conf` Schema-Version + Key-Aware Migrations Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reconcile each project's hand-owned `sandbox.conf` against central on every sync — additive keys flow in automatically, the two-per-lifetime semantic changes are handled by explicit key-aware `migrations/` hooks, and the duplicate-key footgun in the parser is closed.

**Architecture:** A `# schema-version: N` comment marker lives inside `sandbox.conf` (invisible to the `grep "^key="` parser). A git-tracked `migrations/NNN-*.sh` directory holds one idempotent key-only hook per **semantic** change (rename/split/removal only — never a plain addition). A new sourcing-safe `reconcile_sandbox_conf` helper in `sync-to-projects.sh` replaces today's WARN-only branch: it runs pending hooks, additively appends any central keys the project lacks (never touching keys it already has), then unconditionally ensures the marker. Two authoring/CI tools (`bump-sandbox-version.sh`, `check-sandbox-version.sh --check`) and a duplicate-key guard in `get_versions()` round it out.

**Tech Stack:** Bash (`sync-to-projects.sh`, `sandbox-common.sh`, two new standalone scripts, `migrations/*.sh` hooks), git (the CI gate diffs `sandbox.conf` against a base ref). Tests follow the repo's existing style — plain bash under `tests/`, `pass`/`fail` helpers, `mktemp -d` fixtures, sourcing the real library or running the real script and asserting on output (see `tests/test-tools-d.sh` and `tests/test-docs-path.sh`).

## Global Constraints

- **The marker is a plain integer** on a single comment line: `# schema-version: N`. A project file **missing the line is version 0**. Central's current value is **3** (two historical semantic migrations, `002` and `003`).
- **The marker is ensured unconditionally on every sync** — inserted if absent, updated in place if present — regardless of whether any hook fired or any key was added. This is what backfills pre-existing projects with no one-time bootstrap script.
- **A migration hook is added ONLY for a semantic change** (rename / split / removal / in-place meaning change). A plain new `newtool=OFF` key needs **no hook and no version bump** — reconcile backfills it automatically.
- **Hooks touch only `key=value` lines, never comment text**, and must be **idempotent** (check their own precondition, no-op if already applied). They receive the target project's `sandbox.conf` path as `$1`.
- **Reconcile never overwrites a key the project already has**, whatever its value. New keys land under one `# New options synced from upstream (<date>)` banner at the end of the project file — not re-threaded into their original section.
- **Migrations resolve relative to `script_dir`** in `sync-to-projects.sh` (`"${script_dir}/migrations"`).
- **`migrations/` is visible (not `.migrates/`)** — hidden dot-dirs in this repo mean generated/user-local copies; committed infra dirs (`allowlist-domains.d/`, `tools.d/`) are visible.
- **Edit canonical files only.** `CLAUDE.md`, `.github/copilot-instructions.md`, `.kiro/steering/AGENTS.md` are symlinks to `AGENTS.md` — never edit them directly. Never hand-edit the `.ai-containers/` synced copies (regenerated by `sync-to-projects.sh`).
- **Scope is `ai-containers` only.** The spec's "mgd-ai-containers adaptation" section is forward-looking context for a later, separate port — **do not** touch `mgd-ai-containers` in this plan.
- Match the existing bash block style; edits are surgical. Commit after every task. Every commit ends with `Co-Authored-By: Claude Sonnet 5 <noreply@anthropic.com>`.

```bash
# Returns empty string if the key is absent or has no value. Exits with an error
# if the key appears more than once — a duplicated key=line (from a bad manual
# edit or an interrupted reconcile) would otherwise resolve silently to whichever
# occurrence head -1 hits first. This guard makes every caller (build.sh, runme.sh,
# repo.sh) safe.
get_versions() {
  local key="$1"
  local matches
  matches=$(grep -c "^${key}=" "$config_file" 2>/dev/null || true)
  if (( matches > 1 )); then
    printf 'ERROR: duplicate key "%s" (%d occurrences) in %s — remove the extra line(s).\n' \
      "$key" "$matches" "$config_file" >&2
    exit 1
  fi
  local raw
  raw=$(grep "^${key}=" "$config_file" 2>/dev/null | head -1 | cut -d= -f2-)
  # Strip inline comments (e.g. "21 # LTS version" → "21")
  raw="${raw%%#*}"
  # Trim whitespace
  raw="${raw#"${raw%%[![:space:]]*}"}"
  raw="${raw%"${raw##*[![:space:]]}"}"
  printf '%s' "$raw"
}
```

(`grep -c` exits 1 on zero matches; `|| true` keeps it from tripping the sourcing script's `set -e`, and `matches` is then `0`.)

```bash
#!/usr/bin/env bash
# Migration 002: collapse the early per-version openjdk booleans
#   openjdk-21=ON / openjdk-25=OFF / ...
# into a single comma-list key
#   openjdk=<versions that were ON, in file order>
#
# Idempotent (no-op if no legacy openjdk-<N>= key exists). Key-only: comment
# lines and every other key pass through untouched. Receives the target
# sandbox.conf path as $1.
set -euo pipefail
file="$1"

# Precondition: at least one legacy per-version key present.
grep -qE '^openjdk-[0-9]+=' "$file" 2>/dev/null || exit 0

# Collect versions whose legacy key was ON, in file order.
versions=()
while IFS= read -r line; do
  key="${line%%=*}"                                  # openjdk-21
  val="${line#*=}"; val="${val%%#*}"; val="${val// /}"  # strip inline comment + spaces
  ver="${key#openjdk-}"                              # 21
  [[ "$val" == "ON" ]] && versions+=("$ver")
done < <(grep -E '^openjdk-[0-9]+=' "$file")

merged=""
if (( ${#versions[@]} > 0 )); then
  IFS=,; merged="${versions[*]}"; unset IFS
fi

# Rewrite: drop every legacy key line; emit the single openjdk= key where the
# FIRST legacy key was (so it keeps its section position).
tmp="$(mktemp)"
inserted=0
while IFS= read -r line || [[ -n "$line" ]]; do
  if [[ "$line" =~ ^openjdk-[0-9]+= ]]; then
    if (( inserted == 0 )); then
      printf 'openjdk=%s\n' "$merged" >> "$tmp"
      inserted=1
    fi
    continue
  fi
  printf '%s\n' "$line" >> "$tmp"
done < "$file"
mv "$tmp" "$file"
```

```bash
  v="$(grep -E '^# schema-version:[[:space:]]*[0-9]+' "$f" 2>/dev/null | head -1 \
        | sed -E 's/^# schema-version:[[:space:]]*([0-9]+).*/\1/')"
  printf '%s' "${v:-0}"
}

# Ensure FILE ($1) carries '# schema-version: N' ($2): update in place if the
# marker exists, else append it. Unconditional — called every reconcile.
conf_set_version() {
  local f="$1" n="$2"
  if grep -qE '^# schema-version:' "$f" 2>/dev/null; then
    local tmp; tmp="$(mktemp)"
    sed -E "s/^# schema-version:.*/# schema-version: ${n}/" "$f" > "$tmp"
    mv "$tmp" "$f"
  else
    printf '# schema-version: %s\n' "$n" >> "$f"
  fi
}

# Reconcile a project's sandbox.conf ($2) against central ($1) using the hooks in
# MIGRATIONS_DIR ($3):
#   1. run every NNN-*.sh whose NNN > the project's recorded version, ascending;
#   2. additively append any central key the project lacks, under one dated banner
#      (a key the project already has is NEVER touched, whatever its value);
#   3. unconditionally ensure the project's marker matches central's version.
reconcile_sandbox_conf() {
  local central="$1" project="$2" migrations_dir="$3"
  local central_version project_version
  central_version="$(conf_schema_version "$central")"
  project_version="$(conf_schema_version "$project")"

  # 1. Pending migration hooks, ascending numeric order.
  if [[ -d "$migrations_dir" ]]; then
    local hook nnn
    for hook in "$migrations_dir"/[0-9][0-9][0-9]-*.sh; do
      [[ -e "$hook" ]] || continue          # no hooks → glob stays literal
      nnn="$(basename "$hook")"; nnn="${nnn%%-*}"
      # 10#$nnn forces base-10 (leading zeros must not be read as octal).
      if (( 10#$nnn > project_version )); then
        bash "$hook" "$project"
      fi
    done
  fi

  # 2. Additive backfill of central keys the project lacks.
  local added=() key
  while IFS= read -r key; do
    [[ -z "$key" ]] && continue
    grep -qE "^${key}=" "$project" 2>/dev/null || added+=("$key")
  done < <(grep -E '^[A-Za-z0-9_-]+=' "$central" | sed -E 's/^([A-Za-z0-9_-]+)=.*/\1/')

  if (( ${#added[@]} > 0 )); then
    {
      printf '\n# New options synced from upstream (%s)\n' "$(date +%Y-%m-%d)"
      for key in "${added[@]}"; do
        printf '%s=%s\n' "$key" "$(grep -E "^${key}=" "$central" | head -1 | cut -d= -f2-)"
      done
    } >> "$project"
    printf '  Reconciled sandbox.conf: appended %d new key(s): %s\n' \
      "${#added[@]}" "${added[*]}"
  fi

  # 3. Ensure the marker, unconditionally (backfills pre-marker/v0 files).
  conf_set_version "$project" "$central_version"
}
```

```bash
_keys() { grep -E '^[A-Za-z0-9_-]+=' | sed -E 's/^([A-Za-z0-9_-]+)=.*/\1/' | sort -u; }

_ver()  { grep -E '^# schema-version:[[:space:]]*[0-9]+' | head -1 \

# Keys present in the base ref but gone now (removed or renamed away).

if [[ -z "$removed" ]]; then

  echo "       Author the change with ./bump-sandbox-version.sh <slug>, implement the hook, and bump." >&2
  exit 1
fi

printf -v nnn '%03d' "$cur_ver"
if ! ls "${migrations_dir}/${nnn}-"*.sh >/dev/null 2>&1; then
  echo "ERROR: schema-version bumped to ${cur_ver} but no migrations/${nnn}-*.sh hook exists to cover:" >&2

  exit 1
fi

echo "check-sandbox-version: OK (removed/renamed key(s) covered by schema-version ${cur_ver} + hook)."
```
